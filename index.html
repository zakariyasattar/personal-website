Test-Lock

  The idea behind Test-Lock developed after I noticed two things. The first being that javascript has an in-built event handler for when the browser exists Full-Screen mode, and the second was that most of the available testing applications were desktop applications that were very tricky to use and navigate. The idea was very simple: take the user into Full-Screen mode when they start an exam, and when they try to exit Full-Screen mode, give them an alert, warning them that if they tried to exit 5 times in total, the exam would be turned in. If the alert was on screen for more than 10 seconds, automatically terminate the exam. The user was shown a countdown timer on the alert. This was the largest project I had ever done until that point because it required not only implementing this relatively simple front-end feature, but also I had to implement proper, secure testing capabilities and an entire functioning back-end for teachers to create and grade tests. Due to the scale of the project, I was able to attain deep knowledge of working with JS and HTML to make dynamic webpages and skills in database creation and management. I was also forced to learn how to store and organize data locally using JSON and SessionStorage, provide and process authentication, and to perform calculations on large sets of data.

 Both the code and a video demo walkthrough are available on GitHub: Test-Lock

Niles West Bus App

  The Niles West Bus App first came into fruition as an idea proposal submitted to the App Development Club, of which I was the president. The problem was that the school buses would arrive in random spots every day, and there was only one security guard that knew the locations of all the buses. Every day hundreds of students would swamp this security guard asking where their bus was located. It was a big problem with an easy solution. It took me and another student a week before the project was completed, and the day we launched we logged over 350 page views. It was beautiful, to this day I've never worked on a project that has been so simple and has had such an impact on the daily lives of so many people. Once this program became an integral part of the school, I really began to understand the level of impact that was possible through my work as a Computer Scientist.
